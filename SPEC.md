# 맏춤법 검사기 스펙

## 제품 기준

- 사이트명은 `리버스 맞춤법 검사기`다.
- 화면 타이틀은 `맏춤법 검사기`를 유지한다.
- 사용자는 올바른 한국어 문장을 입력하고, 결과 영역에서 바뀐 표현을 문장 안의 밑줄/하이라이트로 바로 확인한다.
- 화면 문구는 짧게 쓴다. 기능을 길게 설명하지 않는다.
- 금지 문구는 `AGENTS.md`의 UX Writing 섹션을 따른다.

## 핵심 플로우

1. 사용자가 원문을 입력한다.
2. `검사하기`를 누르거나 `Ctrl+Enter`로 실행한다.
3. 결과 카드에 바뀐 문장이 표시된다.
4. 바뀐 단어는 문장 안에서 직접 강조된다.
5. 결과 카드 안에서 `바뀐 표현` 목록이 함께 표시된다.
6. 사용자는 복사, 공유, 이미지 저장을 아이콘 버튼으로 실행한다.

## 변환 룰 기준

- 상세한 룰 추가/수정 절차는 `RULE_UPDATE.md`를 따른다.
- 입력이 비어 있으면 원문과 결과를 그대로 반환하고 오류 목록은 비운다.
- 일반 룰은 문맥 룰보다 우선한다.
- 문맥 룰은 `includeContextual: true`일 때만 적용한다.
- 같은 위치에 여러 룰이 잡히면 더 긴 원문 매치가 우선한다.
- 겹치는 룰은 앞쪽에서 선택된 룰을 우선하고 뒤쪽 겹침은 버린다.
- 일반 룰이 하나도 적용되지 않을 때만 fallback 룰을 하나 적용한다.

## UI 기준

- 320px 이상에서 가로 overflow가 없어야 한다.
- 375, 768, 980, 1024, 1280, 1440 폭을 주요 확인 폭으로 본다.
- 아이콘 버튼은 최소 44px 터치 타깃과 `aria-label`/`title`을 가진다.
- 결과 상태 변경 영역은 보조기기용 상태 전달을 제공한다.
- `prefers-reduced-motion`을 존중한다.
- `transition: all`은 쓰지 않는다.

## 배포 기준

- 패키지 매니저는 `pnpm`이다.
- Vercel 빌드 설정은 `vercel.json`을 기준으로 한다.
- 빌드 명령은 `pnpm build`다.
- Vercel output directory는 `dist/public`이다.
- 공유 URL은 기본적으로 현재 접속 도메인을 사용한다.
- 대표 도메인을 강제로 고정해야 할 때만 `VITE_SITE_URL`을 설정한다.

## 검증 하네스

변경 후 가능한 한 아래를 실행한다.

```bash
pnpm qa
```

개별 명령은 다음과 같다.

```bash
pnpm check
pnpm test
pnpm build
```
