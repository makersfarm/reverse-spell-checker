# 맞춤법 룰 업데이트 기준

이 문서는 `client/src/lib/reverseSpellCheck.ts`의 변환 룰을 추가하거나 수정할 때 적용한다.

## 원칙

- 이 앱은 올바른 표현을 자주 헷갈리는 표현으로 바꿔 보여주는 도구다.
- 룰은 “웃기려고 아무렇게나 틀리는 것”이 아니라, 실제로 혼동되는 표현을 다룬다.
- 새 룰은 반드시 테스트 fixture와 함께 추가한다.
- 문맥 없이 오작동할 가능성이 큰 룰은 기본 룰에 넣지 않고 contextual 룰로 둔다.
- 사용자 피드백은 곧바로 반영하지 않고 후보 목록으로 모은 뒤 검토한다.

## 출처 우선순위

1. 국립국어원 한국어 어문 규범
   - 한글 맞춤법
   - 표준어 규정
   - 외래어 표기법
   - https://www.korean.go.kr/kornorms/
2. 국립국어원 표준국어대사전
   - 단어의 표준성, 의미, 활용 확인
   - https://stdict.korean.go.kr/
3. 국립국어원 온라인가나다/공식 해설
   - 특정 표현의 쓰임 확인
   - 단, 답변 맥락을 함께 기록한다.
4. 실제 사용자 피드백
   - 빈도와 재현 문장을 본다.
   - 공식 근거 없이 바로 기본 룰로 올리지 않는다.

## 룰 분류

### 기본 룰

문맥 없이도 비교적 안전하게 적용할 수 있는 룰이다.

예:

- `오랜만에` → `오랫만에`
- `며칠` → `몇일`
- `돼요` → `되요`

기준:

- 일반 문장에서 오작동 가능성이 낮다.
- 원문과 결과가 겹쳐도 더 긴 표현 우선순위로 제어 가능하다.
- fixture 테스트를 통과한다.

### 문맥 룰

문맥에 따라 맞고 틀림이 달라지는 룰이다.

예:

- `든지`/`던지`
- `데`/`대`
- `되`/`돼` 중 일부 활용

기준:

- 기본 옵션에서는 적용하지 않는다.
- `includeContextual: true`일 때만 적용한다.
- UI에서 나중에 “문맥 후보”처럼 별도 표시할 수 있게 유지한다.

### fallback 룰

기본 룰이 하나도 잡히지 않을 때만 결과를 만들기 위한 보조 룰이다.

기준:

- 한 번에 하나만 적용한다.
- 실제 맞춤법 근거보다 “빈 결과 방지” 목적이다.
- 기본 룰보다 앞서 적용되면 안 된다.

## 룰 추가 절차

1. 후보 표현을 기록한다.
   - 올바른 표현
   - 바꿀 표현
   - 분류: `맞춤법`, `띄어쓰기`, `표준어`
   - 이유 문장
   - 공식 근거 URL
2. 문맥 의존 여부를 판단한다.
3. `client/src/lib/reverseSpellCheck.ts`에 룰을 추가한다.
4. `client/src/lib/reverseSpellCheck.fixtures.ts`에 fixture를 추가한다.
5. `pnpm test`로 룰 결과를 확인한다.
6. 가능하면 `pnpm qa`까지 실행한다.

## fixture 작성 기준

fixture는 다음을 검증해야 한다.

- 바뀐 문장에 기대 표현이 포함되는가
- 오류 목록의 `original`이 기대값을 포함하는가
- 오류 목록의 `wrong`이 기대값을 포함하는가
- 오류 타입이 기대 분류와 맞는가
- fallback 룰이면 기본 룰이 먼저 잡히지 않는 입력을 쓰는가

## 거절 기준

아래 후보는 바로 룰로 넣지 않는다.

- 특정 커뮤니티 밈에만 가까운 표현
- 실제 사용 빈도가 낮고 억지스러운 표현
- 공식 근거가 없고 피드백 1건뿐인 표현
- 욕설/비하/차별 표현을 결과로 만드는 룰
- 문맥 없이는 정상 문장을 과하게 망가뜨리는 룰

## 피드백 수집으로 넘어갈 때

나중에 피드백 기능을 붙이면 아래 필드를 최소 단위로 모은다.

- 입력 원문
- 생성 결과
- 사용자가 문제 삼은 표현
- 기대한 바뀐 표현
- 액션: `좋아요`, `별로예요`, `직접 제안`
- 클라이언트 버전 또는 룰셋 버전
- 시간

개인정보가 들어갈 수 있으므로 원문 저장은 별도 동의 또는 익명화 정책을 먼저 정한다.
